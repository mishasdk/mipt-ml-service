# 2. Canary deployment strategy

Date: 2026-05-10

## Status

Accepted

## Context

Нужно выбрать стратегию выката новой версии в production так, чтобы минимизировать риск багов для всех пользователей и иметь возможность быстро откатиться.

## Decision

Используем Canary: новая версия сначала получает небольшую долю трафика (например, 10%), и при стабильных метриках доля растёт до 100%.

## Consequences

**Плюсы.** Баги затрагивают только малую часть пользователей; откат — это просто возврат весов трафика; нет downtime; получаем реальную обратную связь до полного выката.

**Минусы.** Усложняется CI/CD; нужен нормальный мониторинг с метриками и алертами; старая и новая версии работают параллельно, нужно поддерживать обратную совместимость.
